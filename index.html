教程二---部署到github

2020-01-21
Hexo 教程

前言

根据前一章的内容，建立的博客网站只能在本地运行查看，如果想让更多人了解到你的博客，就需要将博客部署到外网可访问的服务器上。对于新手来讲，Github是一个不错的选择，因为Github部署步骤简单且免费。但是国内访问Github速度比较慢。

1 注册 Github 账号

已有 Github 账号跳过此步。

2 创建 repository

repository 相当于一个仓库，用于存放代码文件。登录 Github 账号，并且新建一个 Repository。
注意 新建的仓库名称一定要与 Github 账户名一致

3 配置 SSH

首先查看是否已经存在SSH Keys
打开git bash，输入 ls -al ~/.ssh(如果用过linux,对ls命令应该不会陌生)

ls -al ~/.ssh

由于我已经生成过ssh，因此会出现以下内容

如果不存在也没关系，如果存在的话，直接删除.ssh文件夹里的所有内容

输入以下指令，回车，出现提示的让你输入的时候直接回车就行。邮箱地址是注册 Github 时用的邮箱

ssh-keygen -t rsa -C “邮箱地址”

记事本打开 .ssh/id_rsa.pub 文件，并复制里面的内容，然后打开 Github 主页，进入个人设置（右上角个人图标 -> Settings）

测试是否成功，打开 git bash

ssh -T git@github.com

如果提示 Are you sure you want to connecting(yes/no)?，输入 yes
则会看到以下信息

看到此信息说明 SSH 配置成功！

同时在 git bash 里设置 Github 的 user.name 和 user.email 信息

git config –global user.name “你的 Github 用户名“

git config –global user.email “你的 Github 注册邮箱“

4 配置 _config.yml

用记事本打开配置文件 _config.yml

配置 文件中有关 deploy 的部分

5 部署到 Github

首先安装 Git 部署插件，在博客文件路径下进入 git bash，输入命令：

npm install hexo-deployer-git –save

之后，分别输入三条命令：

hexo clean

hexo g

hexo d

完成之后，若没有出现报错信息（一般按照这个步骤不会报错，如果出现，欢迎交流）

在浏览器下输入 http://xxxx.github.io/,
就会发现你的博客已经上线了

Hexo 教程一---本地搭建

2020-01-21
Hexo 教程

前言

作为一个编程爱好者，一直希望拥有一个自己的博客网站，但是由于时间的关系，一直没有能够动手搭建。在春节期间，闲来无事，开始着手搭建自己的博客。为什么选择hexo，主要是因为Hexo是一个快速、简洁且高效的静态博客框架，适合小白。下面记录了我搭建hexo博客的一些主要步骤。因为我自己也是一个小白，所以搭建流程会更可能的详细

1 环境搭建

  • Node.js
  • npm
  • git

Node.js中已经带有了npm。
Node.js和git的安装请自行百度

完成安装之后，win+r 输入cmd 并打开，依次输入node -v、npm -v 和 git –version 并回车，出现下图版本号则说明安装成功。

2 本地安装 hexo 博客程序

2.1安装Hexo

使用 npm一键安装 Hexo 博客程序

npm install -g hexo-cli

安装时间可能会有点久，由于我已经安装过了，再次运行该命令，会出现如下图所示的界面

2.2本地初始化

在本地新建一个文件夹 Hexo_Blog (文件夹名称随意)。进入该文件夹，右击 -> Git Bash Here。如下图所示

初始化并安装所需软件：

hexo init #初始化

hexo install #安装组件

安装过程

安装之后，新建文件夹中的文件目录结构

完成之后依次输入以下命令，启动本地服务器进行预览

hexo g

hexo s

访问 http://localhost:4000, 出现 Hexo 默认页面，本地博客安装成功！
